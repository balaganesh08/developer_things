Host a Static Website on AWS

Hosting a static website on AWS provides a scalable, secure, and cost-effective solution for your web presence. In this blog post, we will guide you through the process of setting up a static website using Amazon S3, enhancing it with CloudFront for content delivery, securing it with AWS Certificate Manager (ACM), and routing your custom domain using Route 53.

Step 1: Create a Static Website on Amazon S3

1.1 Create an S3 Bucket

First, you need an S3 bucket to store your website files. Here's how you can create one:

Navigate to S3 Console: Go to the AWS Management Console and open the S3 service.

Create a New Bucket: Click on "Create bucket." Enter a unique name for your bucket that matches your domain name (e.g., example.com).

Configure Bucket Settings: Leave the default settings and create the bucket in the same region where you plan to use CloudFront.

1.2 Configure the Bucket for Static Website Hosting

Once the bucket is created, configure it to host a static website:

Enable Static Website Hosting: Select your bucket, go to the "Properties" tab, and click on "Static website hosting."

Set Index Document: Enter index.html as the index document. Optionally, specify an error document like error.html.

1.3 Upload Your Website Content

Now, upload your website files:

Upload Files: Click on "Upload" and add your HTML, CSS, JS, and other files.

Set Permissions: Ensure the files are publicly readable. You can set permissions during upload or after by selecting the files and modifying their permissions.

Step 2: Distribute Content with CloudFront

2.1 Create a CloudFront Distribution

CloudFront enhances the delivery speed of your website by caching your content at edge locations:

Navigate to CloudFront Console: Open the CloudFront service in the AWS Management Console.

Create a New Distribution: Click on "Create Distribution" and choose "Web" for the delivery method.

Configure Distribution:

  • Origin Domain Name: Enter your S3 bucket's website endpoint (e.g., example.com.s3-website-us-east-1.amazonaws.com).
  • Default Cache Behavior Settings: Configure settings like allowed HTTP methods, caching behavior, etc.
  • Distribution Settings: Set other preferences like distribution name, logging, etc.

Create Distribution: Click "Create Distribution" and note the CloudFront domain name provided.

Step 3: Secure Your Website with AWS Certificate Manager (ACM)

3.1 Request a Certificate

To secure your website with HTTPS, request an SSL certificate:

Navigate to ACM Console: Open the AWS Certificate Manager service.

Request a Certificate: Click on "Request a certificate" and choose a public certificate.

Enter Domain Names: Enter your domain name (e.g., example.com).

Validate Domain: Choose DNS validation and follow the instructions to add CNAME records to your Route 53 hosted zone.

Step 4: Route Traffic with Route 53

4.1 Configure Route 53 Hosted Zone

Route 53 will manage your domain's DNS settings:

Navigate to Route 53 Console: Open the Route 53 service.

Create Hosted Zone: If not already created, create a hosted zone for your domain.

Add DNS Records:

  • A Record for CloudFront: Add an A record, select "Alias," and choose your CloudFront distribution from the dropdown.
4.2 Validate ACM Certificate

After adding the DNS records, ACM will automatically validate your domain ownership and issue the certificate.

Step 5: Update CloudFront to Use the ACM Certificate

Go to CloudFront Console: Select your distribution.

Edit Distribution Settings: Under the SSL/TLS settings, add the ACM certificate you requested.

Save Changes: Update the distribution settings to use the new certificate.

Step 6: Test Your Setup

Access Your Website: Open a browser and navigate to your domain to ensure the website loads over HTTPS.

Check for Errors: Resolve any issues if the website does not load correctly or if there are security warnings.
